# SaaS MVP Backend Job Post

## Original Job (Structured Summary)

Client wants a backend developer to build a simple and reliable MVP automation flow for a SaaS product.

- Goal: fast, simple, scalable MVP without over-engineering
- Frontend: already built in Framer (currently static), needs backend connection
- Budget style: hourly, less than 30 hrs/week, less than 1 month
- Priority: reliability, speed, clear communication, potential ongoing work

## Core Flow Requested

1. User buys a pass with Stripe
2. Stripe webhook triggers backend logic
3. Backend creates/stores user data in database (Supabase or similar)
4. Backend generates a dynamic PDF from user inputs
5. System emails PDF automatically
6. System sends unique personal link for user data updates and PDF regeneration

## Required Skills

- Stripe webhooks (must-have)
- SaaS/MVP backend experience
- Pragmatic implementation style (no unnecessary complexity)
- PDF generation
- Email automation

## Nice To Have

- Supabase experience
- Framer integration experience

## Client’s Key Screening Requirement

Client explicitly asks candidates to confirm similar past implementation of:

`Stripe webhook -> backend processing -> PDF generation -> email delivery`
